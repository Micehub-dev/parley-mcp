# Product Brief

## Problem

- 멀티 LLM 토론을 Codex, Claude, Gemini 어느 쪽에서 시작하더라도 동일한 계약으로 재개/종료하기 어렵다.

## Users

- Codex 기반 오케스트레이터 사용자
- Claude Code 사용자
- Gemini CLI 사용자
- 내부 운영자 / 개발자

## Value Proposition

- 벤더 종속 없이 동일한 MCP 계약으로 토론 세션을 관리한다.

## Goals

- orchestrator-agnostic session management
- participant adapter normalization
- workspace-level memory accumulation

## Non-Goals

- 초기 단계에서 완성형 UI 제공
- 벤더별 독자 계약 분기

## Success Metrics

- end-to-end debate success rate
- session resume success rate
- lease conflict recovery success rate
